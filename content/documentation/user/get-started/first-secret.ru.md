После того как вы получили доступ к **Deckhouse Stronghold** и выполнили первый вход, следующий шаг — сохранить и прочитать первый секрет.

Для первого знакомства с Stronghold обычно используется механизм секретов `KV`, который предоставляет хранилище key-value для работы с произвольными секретами.

> Примечание  
> Механизм секретов `KV` может работать в двух режимах:
> - `KV v1` — без версионирования;
> - `KV v2` — с версионированием и дополнительными возможностями.  
> В этом разделе используется нейтральный базовый сценарий. Подробности о различиях между `KV v1` и `KV v2` приведены в соответствующих разделах документации.

## Что нужно до начала

Перед выполнением примеров убедитесь, что:

- вы уже вошли в Stronghold;
- у вас есть токен с правами на запись и чтение в выбранном `mount path`;
- администратор предоставил вам доступ к `KV`-хранилищу.

Если вход уже выполнен, вы можете использовать команды `d8 stronghold` для записи и чтения секретов.

## Шаг 1. Выберите путь для секрета

Секреты в `KV`-хранилище записываются по пути.

Для первого примера можно использовать тестовый путь, например:

```text
secret/my-first-secret
```

Или, если в вашей инсталляции используется другой `mount path`, путь может выглядеть иначе. Используйте значения, согласованные с администратором.

## Шаг 2. Запишите секрет

Сохраните тестовый секрет командой:

```bash
d8 stronghold kv put secret/my-first-secret username=demo password=secret123
```

Если операция выполнена успешно, Stronghold подтвердит запись секрета.

## Шаг 3. Прочитайте секрет

Прочитайте только что сохранённый секрет:

```bash
d8 stronghold kv get secret/my-first-secret
```

В результате вы должны увидеть сохранённые значения.

Пример ожидаемого результата:

```text
====== Data ======
Key         Value
---         -----
password    secret123
username    demo
```

## Шаг 4. При необходимости обновите секрет

Если нужно изменить значение секрета, выполните повторную запись по тому же пути:

```bash
d8 stronghold kv put secret/my-first-secret username=demo password=new-secret
```

После этого прочитайте секрет снова:

```bash
d8 stronghold kv get secret/my-first-secret
```

## Шаг 5. Удалите тестовый секрет

После завершения проверки можно удалить тестовый секрет:

```bash
d8 stronghold kv delete secret/my-first-secret
```

> Примечание  
> Поведение удаления зависит от того, используется `KV v1` или `KV v2`. В `KV v2` удаление может быть мягким и работать через версионирование. Подробности приведены в профильных разделах по `KV`.

## Что происходит при работе с секретом

В базовом сценарии Stronghold позволяет:

- записывать значения по выбранному пути;
- читать сохранённые данные;
- обновлять значения;
- удалять секрет.

Именно так обычно начинается практическая работа с механизмом секретов `KV`.

## Если команда не работает

Если при записи или чтении секрета возникает ошибка, проверьте:

- успешно ли выполнен вход;
- действителен ли токен;
- есть ли у вас права на выбранный путь;
- правильно ли указан `mount path`;
- включён ли соответствующий механизм секретов.

Если проблема сохраняется, обратитесь к администратору Stronghold.
