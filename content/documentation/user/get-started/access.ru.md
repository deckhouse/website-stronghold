## Доступ к проекту

Чтобы начать работу с **Deckhouse Stronghold**, пользователю необходимо получить параметры доступа от администратора и настроить рабочее окружение.

В типовом сценарии для работы из командной строки используется утилита `d8` и аутентификация через OIDC.

## Что нужно получить от администратора

Перед началом работы убедитесь, что администратор предоставил вам:

- адрес сервера Stronghold;
- способ аутентификации;
- подтверждение, что у вашей учётной записи есть необходимые права доступа;
- дополнительные параметры подключения, если они требуются в вашей инсталляции.

Если в организации используется корпоративный identity-провайдер, обычно достаточно получить доступ к нужной группе или роли.

## Подготовка CLI-доступа

Чтобы настроить доступ к своему проекту из командной строки в Deckhouse Stronghold, выполните следующие действия.

### Шаг 1. Установите утилиту d8

Установите [утилиту `d8`](/products/kubernetes-platform/documentation/v1/cli/d8/).

> Примечание  
> Утилита `d8` используется для работы с Stronghold через CLI в сценариях, где Stronghold интегрирован с экосистемой Deckhouse.

### Шаг 2. Укажите адрес Stronghold

Установите адрес вашего сервера Stronghold в переменную окружения:

```bash
export STRONGHOLD_ADDR=https://stronghold.domain.my
```

Замените `https://stronghold.domain.my` на адрес, который вы получили от администратора.

### Шаг 3. Выполните вход

Авторизуйтесь в Stronghold с помощью команды:

```bash
d8 stronghold login -path=oidc_deckhouse -method=oidc -no-print
```

В этом примере используется:

- метод аутентификации `oidc`;
- путь аутентификации `oidc_deckhouse`.

Если в вашей инсталляции используются другие параметры аутентификации, используйте значения, предоставленные администратором.

### Шаг 4. Используйте команды d8 stronghold

После успешного входа используйте следующий формат команд для работы с объектами Stronghold:

```bash
d8 stronghold <command>
```

Например, для чтения состояния сервиса или работы с секретами.

## Как проверить, что доступ настроен правильно

После входа выполните простую команду, которая не изменяет данные, например:

```bash
d8 stronghold status
```

Если команда выполняется успешно, это означает, что:

- адрес Stronghold указан корректно;
- аутентификация выполнена успешно;
- CLI-доступ работает.

## Если вход не выполняется

Если не удаётся получить доступ к Stronghold, проверьте:

- правильно ли указан `STRONGHOLD_ADDR`;
- доступен ли адрес Stronghold из вашей сети;
- используете ли вы правильный метод аутентификации и путь;
- есть ли у вашей учётной записи права доступа;
- не истекла ли ваша сессия аутентификации.

Если проблема сохраняется, обратитесь к администратору Stronghold.
