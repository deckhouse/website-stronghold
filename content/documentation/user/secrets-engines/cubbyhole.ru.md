Механизм секретов `cubbyhole` используется для хранения произвольных секретов в хранилище, привязанном к токену. В `cubbyhole` пути распределены по токенам: ни один токен не может получить доступ к `cubbyhole` другого токена.

Когда срок действия токена истекает, его `cubbyhole` уничтожается.

## Когда использовать Cubbyhole

`Cubbyhole` полезен в сценариях, где нужно:

- временно сохранить чувствительные данные;
- передать данные, привязанные к конкретной сессии или токену;
- работать с секретами, которые не должны быть доступны другим токенам;
- использовать response wrapping и связанные с ним временные данные.

## Чем Cubbyhole отличается от KV

В отличие от механизма секретов `KV`:

- `cubbyhole` привязан к конкретному токену;
- данные в `cubbyhole` недоступны другим токенам;
- время жизни данных определяется временем жизни токена;
- отдельного TTL для самих значений в `cubbyhole` нет.

Также при записи в ключ в `cubbyhole` старое значение полностью заменяется новым.

## Настройка

Механизм секретов `cubbyhole` включён по умолчанию. Его нельзя:

- отключить;
- переместить;
- включить несколько раз.

Это означает, что отдельная активация `cubbyhole` не требуется: он доступен сразу после начала работы с Stronghold.

## Использование

После того как пользователь получил токен с соответствующими правами, он может записывать и читать данные в своём `cubbyhole`.

### Запись данных

```bash
d8 stronghold write cubbyhole/my-secret my-value=s3cr3t
```

Пример результата:

```text
Success! Data written to: cubbyhole/my-secret
```

### Чтение данных

```bash
d8 stronghold read cubbyhole/my-secret
```

Пример результата:

```text
Key         Value
---         -----
my-value    s3cr3t
```

## Что важно учитывать

- Данные в `cubbyhole` изолированы на уровне токена.
- Если токен будет отозван или истечёт, данные в связанном `cubbyhole` будут уничтожены.
- `Cubbyhole` хорошо подходит для временных секретов, но не для общего долговременного хранения.
- Запись по существующему пути перезаписывает старое значение.

## Практические рекомендации

- Используйте `cubbyhole` для данных, которые должны быть доступны только одному токену.
- Не рассчитывайте на `cubbyhole` как на долговременное хранилище.
- Учитывайте срок жизни токена: вместе с ним исчезнут и данные.
- Для общего хранения статических секретов используйте `KV`, а не `cubbyhole`.
