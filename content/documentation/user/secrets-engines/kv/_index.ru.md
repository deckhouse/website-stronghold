Механизм секретов `KV` представляет собой общее хранилище key-value для хранения произвольных секретов в **Deckhouse Stronghold**.

Этот механизм может работать в двух режимах:

- **KV v1** — хранит только одно, последнее значение для ключа;
- **KV v2** — поддерживает версионность, метаданные, мягкое удаление и восстановление версий.

## Когда использовать KV

`KV` подходит, если вам нужно:

- хранить статические секреты;
- сохранять пароли, токены, API-ключи и другие чувствительные значения;
- читать и обновлять секреты по известному пути;
- при необходимости использовать версионность и историю изменений.

## Чем отличаются KV v1 и KV v2

### KV v1

`KV v1` — это более простой режим хранения:

- сохраняется только текущее значение секрета;
- история изменений не хранится;
- операции выполняются быстрее и требуют меньше места в хранилище.

### KV v2

`KV v2` — это расширенный режим:

- у секрета может быть несколько версий;
- доступны метаданные по версиям;
- поддерживаются `CAS`, `patch`, мягкое удаление, восстановление и окончательное уничтожение данных.

## Репликация KV

Для `KV v1` и `KV v2` Stronghold поддерживает репликацию между экземплярами Stronghold в режиме `master-slave` с использованием pull-модели.

Репликация полезна, если нужно:

- автоматически копировать секреты между кластерами;
- построить схему чтения из выделенного кластера-получателя;
- синхронизировать данные между разными пространствами имён и контурами.

> Примечание  
> Репликация поддерживается только между хранилищами одной версии: `kv1` в `kv1` и `kv2` в `kv2`.
