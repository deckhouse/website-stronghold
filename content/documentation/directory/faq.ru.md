На этой странице собраны короткие ответы на частые вопросы о Deckhouse Stronghold и его основных возможностях.

## Что такое Deckhouse Stronghold?

Deckhouse Stronghold — это система для хранения секретов, управления доступом к ним, выпуска временных учётных данных и выполнения криптографических операций через разные механизмы секретов, например KV, Transit, PKI, SSH, LDAP, Kubernetes и базы данных.

## Чем отличается KV v1 от KV v2?

`KV v1` хранит только текущее значение секрета. При записи старое значение заменяется.

`KV v2` поддерживает версионность, метаданные, мягкое удаление, восстановление удалённых версий, полное уничтожение версий и частичное обновление через `patch`.

Если вам нужно хранить только актуальное значение и не нужна история изменений, подойдёт `KV v1`. Если нужны версии, метаданные и более гибкое управление жизненным циклом секрета, используйте `KV v2`.

## Хранит ли Transit мои данные?

Нет. Механизм секретов `Transit` выполняет криптографические операции с данными, но не хранит переданный открытый текст и не хранит возвращённый шифротекст. Хранение данных остаётся на стороне приложения или другой вызывающей системы.

## Можно ли использовать Stronghold для выпуска временных учётных данных?

Да. Deckhouse Stronghold поддерживает выпуск временных credentials для разных сценариев. Например:

- токены Kubernetes для `ServiceAccount`;
- динамические LDAP-учётные данные;
- временные учётные данные для PostgreSQL, MySQL и ClickHouse;
- короткоживущие сертификаты через PKI.

## Можно ли использовать Stronghold как OIDC identity provider?

Да. Deckhouse Stronghold может работать как OIDC identity provider. Для этого настраивается метод аутентификации, клиентское приложение и используется OIDC discovery configuration, включая `issuer`, `client_id` и `client_secret`.

## Может ли Stronghold выпускать OIDC-токены с данными идентичности?

Да. Stronghold может выпускать подписанные JWT, которые соответствуют структуре OIDC ID token. Такие токены создаются на основе роли, связанной с ключом подписи, и могут включать дополнительные claims через шаблон.

## Можно ли автоматически обновлять секреты и токены в приложении?

Да, но способ зависит от сценария. Например:

- `Transit` поддерживает ротацию ключей и `rewrap` без раскрытия открытого текста;
- механизм секретов `Kubernetes` отзывает токены после окончания аренды;
- `PKI` позволяет выпускать сертификаты с ограниченным TTL;
- `LDAP` поддерживает ротацию паролей для статических ролей и наборов учётных записей;
- `KV v2` поддерживает версионность и управление удалением/восстановлением версий.
